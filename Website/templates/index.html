Introduction

  1. Introduction
Welcome to the server side of our P2P file-sharing application. This component functions behind the scenes and is operated via a command-line interface on the terminal, reflecting our commitment to a peer-to-peer architecture. It orchestrates network connections and file directory services without a graphical interface, allowing users to connect, share, and discover files directly.

While the client side is visible through our web interface for ease of access, the server remains the unseen backbone, efficiently managing peer connections and ensuring the integrity of the network.
Server Functionalities

The server script provides the following functionalities:

  • Start Server: Initializes and starts the server on a specified host and port.
  • Listen to Clients: Continuously listens for new client connections and handles them in separate threads.
  • Handle Client: Manages communication with connected clients.
  • Exit: Shuts down the server and closes all client connections.
  • List Clients: Lists all currently connected clients.
  • Ping: A diagnostic tool to confirm the reachability of a client within the network.
  • Discover: Allows clients to query the network, revealing other peers and the files available for sharing.
Client Functionalities

The client script is designed to interact with the server and provides the following functionalities:

  • Connect to Server: Establishes a connection to the server using a specified IP address and port.
  • Send Files: Allows sending files to the server or other connected clients.
  • Receive Files: Can receive files from the server or other clients.
  • Exit: Closes the client's connection to the server and exits the application.
  • Command Input: If the system has a command shell, this enables sending commands directly to the server and viewing responses.
